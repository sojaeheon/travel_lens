# 🗺️ 데이터 기반 글로벌 여행지 시각화 플랫폼 기획서

## 1. 프로젝트 개요

### 프로젝트명: **트래블 렌즈 (Travel Lens)**  
데이터 기반 글로벌 여행 인사이트 플랫폼

---

### 추진 배경
- 해외여행 수요는 꾸준히 증가하고 있으나, 여행객들은 수많은 정보 속에서 최적의 여행지를 선택하는 데 어려움을 겪고 있습니다. (**정보 과부하**)
- 여행지 관련 정보(안전, 인기, 비용)가 여러 플랫폼에 분산되어 있어, 이를 통합적으로 비교하고 분석하는 데 많은 시간이 소요됩니다.

---

### 프로젝트 목적
- 전 세계 여행지의 **인기**, **위험도**, **트렌드**를 세계지도 상에서 직관적으로 시각화
- 특정 여행지를 선택할 시, 다음과 같은 데이터를 기반으로 대시보드를 제공  
  - 실시간 기사  
  - 블로그  
  - 항공료  
  - 환율  
  - 인구 유입량  
- 복합적인 정보를 하나의 UI로 제공하여 **데이터 기반 여행 의사결정**을 지원

---

## 2. 대상 고객 (Target Audience)

### 핵심 타겟
- **여행 계획 초기 단계의 사용자**  
  막연하게 해외여행을 가고 싶지만, 구체적인 목적지를 정하지 못한 탐색 중인 사용자

### 서브 타겟
- **데이터 기반 의사결정 선호 사용자**  
  감성 후기보다 객관적 데이터(비용, 안전, 트렌드)를 중요하게 생각하는 사용자
- **여행 트렌드 팔로워**  
  현재 뜨는 여행지 및 최신 이슈에 민감한 사용자

---

## 3. 주요 기능 (Core Features)

---

### F-01: 인터랙티브 글로벌 맵 (Interactive Global Map)

#### 기능
- **세계지도 UI:** 직관적인 지도 기반 탐색 제공
- **지도 검색:** 국가/도시 검색 및 빠른 정보 접근
- **인기 여행지 시각화:**  
  - 클릭량, 유입량, 검색량  
  - 입국자 수  
  - 좋아요 수  
  → 히트맵 또는 버블 차트로 시각화
- **위험 지역 시각화:**  
  외교부 여행경보 + 현지 뉴스(치안/재난) 기반 색상 표시
- **개인화 기능:** 사용자가 '찜(좋아요)'한 여행지를 지도에 별도 표시

#### 사용 기술
- **Frontend:** Vue.js  
- **Backend:** Django  
- **Database:** PostgreSQL  

---

### F-02: 여행지 상세 인사이트 대시보드 (Destination Insight Dashboard)

#### 기능
- 국가/도시 클릭 시 상세 정보 패널 활성화
- **실시간 뉴스 & 블로그 분석:**  
  - 관련 최신 기사  
  - 실시간 블로그 피드  
  - Elasticsearch 기반 키워드 분석
- **환율 정보:** 실시간 환율 그래프 제공
- **항공료 변동 분석:** 과거 + 실시간 항공료 추이 시각화


#### 사용 기술
- Vue.js, Django, Elasticsearch, Flink/Kafka (실시간 처리)

---

### F-03: AI 에이전트 기반 맞춤형 추천 (AI Agent Recommendation)

#### 기능
- 자연어 기반 여행지 추천  
  예)  
  - “12월에 따뜻하고 저렴한 곳 추천해줘”  
  - “치안이 좋고 볼거리가 많은 여행지는?”
- 축적된 데이터 기반 최적 여행지 추천 + 대시보드 링크 제공
- Elasticsearch 기반 데이터 검색 활용

#### 사용 기술
- AI Agent (LLM 연동), Django API, Elasticsearch  

---

### F-04: 글로벌 트렌드 및 커뮤니티 (Global Trends & Community)

#### 기능
- **글로벌 채팅 기능 (Travel Talk):**  
  실시간 여행 정보 교환
- **채팅 워드 클라우드:**  
  Elasticsearch 기반 키워드 분석
- **인기 국가/도시 Top 10:**  
  Spark 배치 분석을 통한 주간/월간 랭킹
- **주요 환율 제공:**  
  - USD  
  - JPY  
  - EUR  
  - GBP  
  - CNY  
  - AUD  

#### 사용 기술
- Django Channels, Vue.js  
- Elasticsearch (채팅 분석, 워드 클라우드)  
- Spark (랭킹 집계)

## 4. 기술 스택 및 시스템 아키텍처

아래는 대용량 실시간 데이터 처리를 고려한 아키텍처 구성요소입니다.

| 구분 | 기술 스택 | 역할 |
|------|-----------|------|
| **Frontend** | Vue.js | UI/UX, 인터랙티브 맵, 대시보드 시각화 |
| **Backend (API)** | Django | REST API, 인증, AI 에이전트 연동, 채팅 서버 |
| **Data Ingestion** | Apache Kafka | 실시간 데이터 스트림 수집/버퍼링 |
| **Real-time Processing** | Apache Flink | 실시간 인기 급상승 감지, 채팅 키워드 분석 |
| **Batch Processing** | Apache Spark | 월별/분기별 여행 트렌드 분석, 인기 랭킹 생성 |
| **Data Lake** | Apache Hadoop (HDFS) | 원본 데이터 및 분석 결과 저장 |
| **Search Engine** | Elasticsearch | 뉴스/블로그/채팅 데이터 색인 및 검색 |
| **Workflow Management** | Apache Airflow | 파이프라인 스케줄링 및 자동화 |
| **AI/ML** | AI Agent | 자연어 질의 처리 및 추천 |
| **Database** | PostgreSQL | 사용자 정보, 찜 목록, 메타데이터 저장 |

## 5. 기대 효과

### 사용자 측면
- 직관적인 시각화 자료를 통해 **여행지 탐색 및 의사결정 시간을 단축**
- 복합 데이터를 기반으로 **합리적인 여행 계획 수립** 가능

### 플랫폼 측면
- 분산된 여행 정보를 모아 **‘여행 데이터 허브’** 구축
- 축적된 데이터로 **맞춤형 광고, 여행 상품 중개 등 수익모델 확장** 가능
